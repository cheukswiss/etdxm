# imperial_direct 临时通信授权——完整协议

## 触发条件

- 皇上明确下达 `imperial_direct` 指令，且指定了目标 Agent
- 太子确认指令合法性（来源为皇上本人）后激活临时授权

## 授权范围

- 太子（Lead）获得向**被指定目标 Agent**发送 SendMessage 的临时权限
- 目标 Agent 获得向太子直接回报执行结果的临时权限（绕过尚书省中转）
- 授权仅限太子与被指定 Agent 之间的双向通信，不扩展至其他六部或三省
- 目标 Agent 可临时超出常规权限执行皇上指令中要求的操作
- 本临时授权不构成对 governance-core 通信权限矩阵本体的修改

## 时效

- 授权自太子转发 `imperial_direct` 指令时生效
- 授权于以下任一条件满足时终止：
  1. 目标 Agent 回报执行完毕
  2. 皇上明确撤销指令
  3. 超时未完成——默认超时 **30 分钟**；太子转发指令时可附加自定义时限（`timeout_minutes` 字段），上限不超过 **60 分钟**；超过 60 分钟的授权需求须由皇上重新下达 `imperial_direct`
- 超时到达后，太子须向目标 Agent 发送授权终止通知，目标 Agent 须立即停止执行并回报当前进度
- 授权终止后，太子与目标 Agent 之间的通信权限自动恢复为矩阵默认值（—）

## 恢复与备案

- 授权终止后，太子须将本次 `imperial_direct` 的完整通信记录（指令内容、目标 Agent、授权时间、终止时间、执行结果）同步至尚书省备案
- 尚书省须在备案后通知门下省，门下省可对本次临时授权进行事后审计
