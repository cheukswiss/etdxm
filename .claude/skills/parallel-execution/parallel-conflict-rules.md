# 并行执行冲突判定细则与约束清单

> 注：创建链超时与整体超时体系的关系，精确数值权威源见 `fault-tolerance/fault-tolerance-tables.md`

## 各部门并行/串行判定细则

| 部门 | 可并行 | 必须串行 |
|------|--------|---------|
| 🔧 工部 | 不同文件/模块 | 同一文件、有函数调用依赖 |
| 📝 礼部 | 不同文档文件 | 同一文档、有交叉引用需同步 |
| ⚖️ 刑部 | 不同模块的测试/审查 | 集成测试（需全部代码就绪） |
| ⚔️ 兵部 | 不同服务/环境 | 同一服务的配置变更 |
| 💰 户部 | 不同数据表/数据源 | 同一表的 schema 变更与数据迁移 |
| 📋 吏部 | 不同 Agent 的配置 | 全局权限规则变更 |

## 完整约束清单

1. **员外郎上限**：单部门同时不超过 **3 个**，超出排队
2. **必须隔离**：员外郎使用独立 worktree，禁止同目录并行修改
3. **员外郎不互通**：所有交互经堂官中转
4. **失败处理**：员外郎失败时堂官按容错规则处理（重试/替换/上报）
5. **创建审批**：员外郎 Teammate 的创建须经尚书省确认
6. **数量上限**：单部门不超过 3 个，全局不超过 12 个
7. **刑部审查**：堂官合并员外郎产出后，整体产出经正常流程送刑部审查。刑部驳回时，由尚书省退回堂官，堂官按需重新启动对应员外郎修复
8. **创建链超时**：员外郎创建三级链各有超时阈值：
   - 堂官 → 尚书省（审批请求）：**30 秒**（审批为轻量决策）
   - 尚书省 → 太子（创建请求）：**60 秒**
   - 创建链总超时：**3 分钟**

   超时降级策略（按超时发生层级）：
   - 堂官 → 尚书省超时：堂官放弃申请，改为串行执行所有子任务，事后补报尚书省
   - 尚书省 → 太子超时：尚书省指示堂官串行执行，不创建员外郎
   - 太子创建后未回报（总超时）：触发 fault-tolerance 尚书省协调超时路径

   > 注：尚书省作为创建链中间节点时，其自身 2 分钟协调超时（fault-tolerance）豁免计入创建链等待时长，以总超时 3 分钟为准。
