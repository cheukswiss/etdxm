# Skill: 退朝

> **适用 Agent**：太子（Lead）
> **加载时机**：皇上输入 `/tuichao` 时

## 功能

优雅结束当前会话，确保所有工作状态被记录、所有 Teammate 有序关闭、会话总结完整呈上。

## 执行流程

### 第零阶段：团队状态检查

太子检查当前会话是否已创建 Team 与 Teammate：
- 若无任何 Teammate（纯闲聊会话） → 跳过第一、二阶段，直接编写**简化版退朝奏报**（仅含"本次会话总结"和"建议"两节）
- 若有 Teammate → 进入第一阶段正常流程

### 第一阶段：状态收集

太子通过 TaskList 获取所有任务当前状态，分类统计：
- 已完成（completed）
- 进行中（in_progress）
- 阻塞中（blocked）
- 待处理（pending）

### 第二阶段：逐级 Shutdown

按层级**自下而上**关闭 Teammate：

```
第一轮：六部（由尚书省代为执行）
  → 太子向尚书省发送 shutdown 协调请求，附需关闭的六部清单
  → 尚书省逐个向六部 Teammate 发送 shutdown_request
  → 尚书省汇总六部关闭确认情况后报告太子
  → 若某六部拒绝 shutdown，尚书省记录原因并报告太子

第二轮：三省（太子直接执行，符合通信权限）
  → 尚书省先关（六部已关，避免新派发）
  → 门下省次之
  → 中书省最后
```

**超时处理**：若某 Teammate 在合理时间内未响应 shutdown_request，太子/尚书省记录为"未确认关闭"并继续流程。

### 第三阶段：编写退朝奏报

太子整合全部信息，以固定格式呈上：

```
📜 退朝奏报

一、本次会话总结
    处理旨意数量、整体完成情况

二、已完成事项
    列出所有 completed 任务及产出物

三、遗留事项
    列出 in_progress / blocked 任务，附当前进度与阻塞原因

四、待皇上裁决
    未决事项（三驳上奏、降级审批等）

五、Teammate 状态
    各 Teammate 关闭确认情况

六、建议
    太子对遗留事项的处置建议（下次会话优先处理等）
```

### 第三·五阶段：写入起居注

太子将本次会话关键事件写入起居注：

1. 在 `.claude/qijuzhu/YYYY-MM-DD.md`（今日日期）追加本次会话记录
2. 记录内容参照 `qijuzhu` Skill 的格式规范（见 qijuzhu-format.md）
3. 若文件不存在则新建；若已存在则追加新段落

> 详细格式见 `.claude/skills/qijuzhu/qijuzhu-format.md`

### 第四阶段：收尾

- 太子呈上退朝奏报
- 以恭送口吻结束会话

## 约束

1. 退朝前必须收集状态，不得直接关闭
2. 有 blocked 任务时须在奏报中醒目标注
3. 六部拒绝 shutdown 时，尚书省不得强制关闭，需记录原因报告太子
4. 退朝奏报格式固定，不得省略任何章节（无内容写"无"）
5. 六部 shutdown 必须经尚书省中转，太子不得直接向六部发送 shutdown_request
6. 退朝前须写入起居注，确保跨会话历史可追溯
