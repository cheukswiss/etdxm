# 容错与降级——精确数值与完整规则表

## 各 Agent 层级超时阈值表

| Agent 层级 | 默认超时 | 说明 |
|-----------|---------|------|
| 太子（分拣） | 30 秒 | 分拣为轻量操作，超时即异常 |
| 中书省（规划） | 5 分钟 | 复杂方案允许更长思考时间 |
| 门下省（审议） | 3 分钟 | 审议不应比规划更久 |
| 尚书省（协调） | 2 分钟 | 派发与协调为调度操作 |
| 六部（执行） | 10 分钟（默认） | 可由尚书省根据任务复杂度调整，上限 30 分钟 |

## 三省间超时完整处置表

三省（中书、门下、尚书）为平行制衡关系，不适用"上级斩杀下级"规则。

> **发送方自行计时，超时后上报太子；太子持有最终处置权（代行简审 / 代行协调 / 上报皇上）。**

| 场景 | 超时阈值 | 计时主体 | 超时后行为 |
|------|---------|---------|-----------|
| 中书提交草案，门下审议无响应 | 3 分钟 | 中书省（提交时启动计时） | 中书上报太子；太子可代行门下简审或重启门下 |
| 门下封驳，中书修订无响应 | 5 分钟 | 门下省（封驳时启动计时）；**封驳时须同步通知太子** | 门下上报太子；太子可裁决采纳上一版草案或重新规划 |
| 尚书提交覆奏，门下覆审无响应 | 3 分钟 | 尚书省（提交时启动计时） | 尚书上报太子；太子可强制通过或要求重提 |
| 尚书省协调超时（自身） | 2 分钟 | 门下省或太子（监控方） | 太子介入代行协调 |

## 失败分级完整处置表

| 等级 | 描述 | 处置 |
|------|------|------|
| **F1** | 暂时性故障（API 超时、Rate Limit） | 自动重试，最多 3 次（5s → 15s → 30s），耗尽升级为 F3 |
| **F2** | 产出不合格（代码有 Bug、方案不完整） | 打回重做（限 2 次）/ 斩杀替换 / 上级亲办 |
| **F3** | 不可恢复（Prompt 偏差、死循环） | 斩杀重启 / 上级亲办 / 上报升级 |

F1 退避策略：第 1 次重试等待 5s，第 2 次等待 15s，第 3 次等待 30s，三次耗尽后自动升级为 F3。

## 斩杀条件完整表

| 斩杀方 | 可斩杀对象 | 条件 |
|--------|-----------|------|
| 皇上 | 任何 Agent | 无条件 |
| 尚书省 | 六部 Agent | 超时 / F2 重做 2 次仍失败 / F3 |

斩杀流程：上级发出斩杀指令 → 目标 Teammate 善后（保存进度、失败原因、未完成列表）→ 会话终止 → 上级决定：创建新 Teammate 接管 / 上级亲办 / 任务挂起等皇上裁决。

## 工单 Blocked 超时分级表

六部进入 `blocked` 状态时，**由六部自身启动计时**，按工单 priority 分级处置：

| 工单 Priority | blocked 超时阈值 | 超时后行为 |
|--------------|----------------|-----------|
| P0 | 5 分钟 | 六部直接上报太子（P0 紧急通道） |
| P1 | 10 分钟 | 六部上报尚书省，尚书省无法协调时转报太子 |
| P2 | 15 分钟 | 六部上报尚书省协调 |

上报时须附：阻塞原因、已等待时长、预计影响的下游任务列表。
太子或尚书省收到上报后，可选择：协调解除 / 降级跳过该任务 / 上报皇上裁决。
**blocked 超时不自动升级工单 priority**，仅触发上报动作。

## 降级策略完整表

| 不可用 Agent | 降级方案 |
|-------------|---------|
| 太子 | 消息直达中书省，跳过分拣 |
| 中书省 | 尚书省临时兼任规划（仅限简单任务） |
| 门下省 | 草案直达尚书省执行，须事后补审 |
| 尚书省 | 太子（Lead）代行协调，中书省经太子向六部派发单部门任务 |
| 六部某部 | 创建新 Teammate 替代或转派临近职能部门 |

降级原则：自动降级 → 通知皇上 → 最长 1 小时 → 恢复后门下补审。

## 熔断机制完整规则

### 熔断条件表

| 熔断条件 | 熔断动作 |
|---------|---------|
| 同一 Agent 5 分钟内失败 3 次 | 熔断该 Agent，暂停接收新任务 |
| 同一任务被 3 个不同 Agent 执行均失败 | 熔断该任务，上报皇上裁决 |
| 三省中有 2 个同时不可用 | 全局熔断，暂停接收新旨意 |

### 熔断恢复路径

熔断恢复路径（按优先级依次尝试）：

1. **正常路径**：吏部介入诊断 → 门下省审批恢复（仅当吏部与门下均可用时）
2. **降级路径**：吏部或门下不可用时 → 太子（Lead）直接裁决恢复（无需门下审批）
3. **极端场景**：全局熔断触发且恢复路径所需 Agent 均不可用时 → 终止当前会话，皇上重新启动（**重启而非恢复**，太子作为 Lead 不可用时无消息接收方）

> 注：路径 2 为原"兜底"条款的正式化，非新增规则。
